Blog #6: Week 12

This week has been the most confusing for me because we are learning React for JavaScript. I am still stuck on a few things like changing properties, but I have looked online and got a better understanding for React while learning new things. I saw that we could use try & catch statements just like in Java. All you need to do encapsulate the code you want to test in a try block and then add a catch statement which will run the code in its block if there is an error present. Error boundaries are a new a new thing in React and makes try & catch statements so much easier. Error boundaries are described as in this website “Error boundaries are react components that catch errors anywhere in the component tree, log them, and can display a fallback UI instead of the component tree that crashed.” They can be written as easy as: 


<ErrorBounday>
    <MyWidget />
<ErrorBoundary />

I learned another thing called React.useRef. In class, I was taught that React.useState([]) is used the assign the current value of the component. React.useRef on the other hand is used to retain the previous value(s) of said component. This is because the previous component will be assigned by React.useEffect(), it works because after components finish rendering React.useEffect() runs.
