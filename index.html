Blog: Week 5

Throughout this week, I have learned many small thing including tricks, purposes, and different formats for the content. CSS you used for appearance of your website and HTML is the format or semantics of the website. I learned about the screen reader and how they help people with visual inability. Programmers can write in id names and alt names for images incase a person would like to hear what they would see on the webpage. For example, writing out 

<img id="Ziggy" src="Ziggy.jpg" alt="Ziggy Stardust">

would show an image, Ziggy.jpg, but would read out Ziggy Stardust in alt="". Another interesting thing was the input for a color in a .CSS file. You could write in a default name such as red or blue; write in rgb(x, x, x); or write #ffee11 (3 hex-decimal numbers), there might be another way but those are the ones I remember.


Another thing I learned was that the format of a website can be easily changed by using flex. A few features by flex would include flex–direction, justify/align–content, and flex–wrap. In order to use flex, you would need to set the display of an element(s) to flex in the .CSS file. Flex–direction allows for the corresponding element(s) to be placed into column or row "box." Justify–content allows for the element(s) to align horizontally at the left or right side of the website, equal space between element(s), or space inserted before and after each element(s), align–content is the same but vertically. Lastly, flex–wrap is generally used for images wrap around page when it touches the end because if there is enough images in row some of them would cut through the website and not be visible.
