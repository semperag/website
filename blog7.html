Blog #7: Week 13

This week I have looked for new knowledge in the React JS area. The first I learned was about portals. These portals allow for components inside React to become DOM nodes separate from the original application that it is in. As this website describes “any event that gets triggered inside the portal will bubble up through the parent in React, rather than into the portal container element itself.” The portal can also be returned by the render function. If you wanted to display some content, you would be able to put the content inside of this component and still display in the other element. 


ReactDOM.createPortal(
    this.props.children,
    document.getElementById(‘modal’),
);

I also learned that there are tools out there that let you set out rules for your code to abide by. These tools are called linters and will highlight code that do not follow your rules. It is very helpful with working in groups, so code quality looks about the same for everyone. I do not know if I have learned this in class, but a fragment is written as <React Fragment>”content”</>. You are able to write this even shorter and just type <></>. And lastly you can call a functional component just like a regular function. For example: 


<SomeComponent/> => SomeComponent()
